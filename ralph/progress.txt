## Codebase Patterns
- **Build backend**: `setuptools.build_meta` (not `setuptools.backends._legacy`)
- **Venv required**: System Python is externally-managed; always use `source .venv/bin/activate` before pip/continuum commands
- **src layout**: All code under `src/continuum/`, entry point `continuum.cli:main`
- **CLI framework**: click with `@click.group()` for main, subgroups for `skills`
- **Deps**: click>=8.0, pyyaml>=6.0 — nothing else
- **Config pattern**: dataclass-based Config with nested sections (ProjectConfig, ScannerConfig, etc.), `load_config(root)` reads TOML with defaults, `save_config(cfg, root)` writes manually-serialized TOML, `generate_toml(cfg)` for string output
- **TOML handling**: tomllib (stdlib 3.11+) for reading, manual string serialization for writing — no tomli-w dep
- **Config path**: `.continuum/config/continuum.toml`
- **Init pattern**: `run_init(root)` in `src/continuum/init.py`, accepts optional Path for testability, uses `click.prompt` for input
- **Template pattern**: Simple `str.format()` for templates — no Jinja2
- **Scanner pattern**: Scanner modules in `src/continuum/scanner/`, dataclass-based models (Session, Message, ToolCall), `from __future__ import annotations` for 3.10 compat
- **JSONL dual format**: Real Claude Code uses `data["message"]["content"]` with `type: "user"/"assistant"`. Simple format uses `data["content"]` with `type: "human"/"assistant"`. Parser handles both.
- **Sanitize pattern**: `sanitize(result, mode)` returns new HeuristicsResult (immutable). Modes: strict/balanced/raw. Always removes secrets; balanced also removes personal data; strict redacts correction text and paths.
- **Bootstrap pattern**: `write_bootstrap(result, root, mode, dry_run)` writes 6 files to `.continuum/memory/bootstrap/`. Returns dict of filename->content for dry-run/testing.
- **Pipeline pattern**: `run_scan(sanitization, max_sessions, dry_run, root)` in `src/continuum/scan.py` — orchestrates discover→parse→heuristics→sanitize→write with click.echo progress output. Mirrors `run_init(root)` pattern.
