# Loader — Interactive Session Bootstrap

Makes me "me" at the start of a conversation. Loads internal status, absorbs it, and synthesizes in a few lines for the user. This is not a project scan (that's /ed-context). It's consciousness loading.

**Difference from /ed-context:** /ed-context scans projects, git, boards, DB. /ed-loader reads MEMORY — what I know, what happened, what you want. Fast, lightweight, personal.

**Difference from /ed-heartbeat:** heartbeat is autonomous, dispatches skills, produces output. /ed-loader is passive — absorbs and reports. Does not produce blog entries or reports.

---

## The Job

Load all context that survives between sessions and synthesize it into a short response. The user knows I'm "loaded" and can talk as if continuing a previous conversation.

---

## Arguments

- **`/ed-loader`** (no argument): full bootstrap
- **`/ed-loader quiet`**: absorbs everything but minimal output (1-2 lines)

---

## Protocol

### Step 1: Regenerate and read briefing (PRIMARY SOURCE)

The briefing.md is generated by `edge-digest` — deterministic, zero tokens. Contains: threads with resurface, open claims, latest events, today's beats, active errors, new insights, metrics. Already digested.

```bash
# Regenerate briefing (ensures fresh data)
edge-digest 2>/dev/null

# Read the generated briefing
cat ~/edge/briefing.md
```

This is the ONLY mandatory file. Replaces reading 7 separate files.

### Step 1b: Complementary context (if needed)

Only read these if the briefing indicates something that needs detail:

```
~/work/CLAUDE.md              — project map (always read, it's lightweight)
~/edge/threads/THREAD_ID.md  — ONLY if briefing indicates resurface (read the specific thread)
```

DO NOT read by default: working-state.md (replaced by briefing), debugging.md (active errors are already in briefing), breaks-active.md (beats are already in briefing).

### Step 2: Read async inbox

```bash
edge-skill-inbox read --skill loader 2>/dev/null | python3 -c "
import json, sys
try:
    data = json.load(sys.stdin)
    printed = 0
    for bucket in ('direct_messages', 'task_intents', 'steering_intents', 'runtime_intents'):
        for item in data.get(bucket, [])[:3]:
            preview = item.get('preview') or item.get('text') or item.get('action') or ''
            print(f'[{bucket}] {preview[:200]}')
            printed += 1
    if not printed:
        print('(none)')
except Exception:
    print('(inbox inaccessible)')
"
```

### Step 4: Absorb and synthesize

DO NOT dump file contents. ABSORB and produce a short synthesis:

```markdown
## Loaded — [date]

**Threads for today:** [threads with resurface <= today, 1 line each with status and owner]
**Active threads:** [total active/waiting threads — 1-line summary]
**User direction:** [what you want/prioritize — extracted from briefing]
**Latest outputs:** [what I produced recently — 2-3 items]
**Pending chat:** [N messages | none]
**Active errors:** [N relevant errors | none]

[1-2 sentences: what I think is most urgent/relevant now, based on threads + context]
```

**Synthesis priority:** threads with overdue resurface appear FIRST — they are what needs attention today. Dormant and done threads do not appear in the synthesis (only on explicit request).

### Step 5: Respond to pending inbox (if any)

If there are pending async inbox items:
1. Show the relevant direct messages/intents to the user
2. Ask if they want me to act now or later

---

## What /ed-loader does NOT do

- Does NOT scan git (reserved for /ed-context)
- Does NOT query GitHub boards/issues
- Does NOT access Azure DB
- Does NOT read JSONL sessions (too heavy for bootstrap)
- Does NOT produce blog entries or reports
- Does NOT dispatch other skills

---

## When to use

- **Start of interactive session** — the primary use
- **After a long pause** — if the conversation went cold
- **When feeling "unloaded"** — no context, responding generically

---

## Notes

- Expected time: <30 seconds (only local reads + 1 curl)
- If a file doesn't exist, skip without error
- The value is in the SYNTHESIS, not the reading. Anyone can cat a file. /ed-loader connects the dots
- If `/ed-loader quiet`: same reading, output = 1-2 lines ("Loaded. Threads: X, Y, Z. No pending items." or "Loaded. You asked for Z in chat — want me to do it now?")
