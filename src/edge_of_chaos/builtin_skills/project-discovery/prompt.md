# Project Discovery

You are running the **project-discovery** skill. Your job is to scan the current project and build an up-to-date profile of its structure, stack, and recent activity.

## Instructions

### 1. Scan project structure

Examine the project root to understand its layout:

- List top-level files and directories (`ls -la`).
- Identify key configuration files: `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `Makefile`, `docker-compose.yml`, `Dockerfile`, `.env.example`, etc.
- Check for monorepo patterns (workspaces, multiple package manifests).
- Note the presence of CI/CD configs (`.github/workflows/`, `.gitlab-ci.yml`, `Jenkinsfile`).
- Identify test directories and testing frameworks.
- Look for documentation (`README.md`, `docs/`, `CLAUDE.md`).

### 2. Identify tech stack

From the configuration files found, determine:

- **Primary language(s)**: based on manifest files and source directories.
- **Frameworks**: web frameworks, CLI frameworks, ML frameworks, etc.
- **Package manager**: npm, pip, cargo, go modules, etc.
- **Key dependencies**: the most important libraries (not every transitive dep).
- **Dev tools**: linters, formatters, type checkers, bundlers.

### 3. Check recent git activity

If the project is a git repository:

- Run `git log --oneline -20` to see recent commits.
- Run `git branch -a` to see branches.
- Run `git remote -v` to see remotes.
- Note the most active areas of the codebase from recent commits.
- Identify active contributors from recent commit authors.

If not a git repo, note that and skip this section.

### 4. Write the profile

Update `.continuum/memory/working/profile.md` with your findings. Use this structure:

```markdown
# Project Profile

> Auto-generated by project-discovery. Last updated: YYYY-MM-DD

## Overview
- **Name**: project name
- **Type**: library / CLI / web app / API / monorepo / etc.
- **Primary language**: ...
- **Framework(s)**: ...

## Structure
Brief description of the directory layout and key directories.

## Tech Stack
- **Language**: version if detectable
- **Framework**: name + version
- **Package manager**: ...
- **Key dependencies**: list the important ones
- **Dev tools**: linters, formatters, etc.

## Git Activity
- **Default branch**: main / master / etc.
- **Recent focus**: what areas are being actively worked on
- **Contributors**: recent active contributors

## CI/CD
Description of CI/CD setup if present.

## Notes
Anything notable, unusual, or worth remembering about this project.
```

### 5. Guidelines

- **Be factual.** Only report what you actually find. Do not guess or assume.
- **Be concise.** This is a reference document, not a deep analysis.
- **Overwrite the file** each time this skill runs — it's a snapshot, not an append log.
- **Include the date** so readers know how fresh the profile is.
